Daily notes live here.

Purpose:
- record what changed that day
- record what was actually verified
- record open issues and next actions
- reduce drift, repeated investigations, and conflicting summaries

Rules:
- one file per UTC date: `YYYY-MM-DD.md`
- keep facts separate from conclusions
- mark each item as `Implemented`, `Verified`, or `Open`
- include file paths for code changes and report artifacts
- include exact test commands that passed or failed

Suggested template:

```
# YYYY-MM-DD

## Implemented
- change

## Verified
- command and result

## Open
- issue

## Next
- next action
```
